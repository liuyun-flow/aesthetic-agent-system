# V2.3.0 发布说明

## 关于本版本

V2.3.0 来自用户实测反馈，围绕两件事：**让收集案例更顺手，让作品描述更完整**。

完整逐项变更见 [CHANGELOG](CHANGELOG.md)；本文件只讲重点。

## 本版本做了什么

### 功能 1：一键收入案例库（预填 + 确认）

- 训练结果区和历史记录详情新增「收入案例库」按钮
- 自动用本次作品 + AI 评估填好案例表单：标题、审美等级、目标用户、价格带、高级/廉价感来源、值得学习点
- **审美等级按 AI 分数自动推导**（≥75 高 / 45-74 中 / <45 低）
- 采用「预填 + 确认」而非盲插入——保存前可核对修正，保护案例库质量
- 训练记录新增 `image_id`，所用图片随案例一并带入

### 功能 2：描述质量优化

- **Vision 自动描述新增商业语境推测**：设计品类 / 目标用户 / 价格带 / 使用场景，标注「AI 推测」，信息不足返回 null（不编造）
- **描述完整度进度条**：提交前提示缺少哪些关键信息
- **引导式补全字段**：品类 / 目标用户 / 价格带 / 使用场景，可一键采用 Vision 推测；提交时并入作品描述，让智能体获得商业语境，判断更准

## 发布形态

与既往一致，**本地部署版（Local Release）**：

- ✅ 用户下载 zip → 解压 → Docker Desktop 启动 → 配置 API Key → 使用
- ❌ 不是 SaaS / 不做登录 / 不做云端存储

## 系统要求

- Docker Desktop（必须）
- DeepSeek API Key（必须）
- OpenAI API Key（可选，用于 Vision 自动描述和语义搜索）

> 说明：本地使用无需任何代理。开发者向 GitHub 推送代码时若在受限网络下可能需要代理，但这与最终用户的部署和使用无关。

## 从 V2.2.x 升级

```bash
git pull origin main
docker compose up --build -d
```

`image_id` 列通过启动时自动迁移添加，旧数据安全无需手动处理。升级前建议先导出备份（设置页 → 数据管理 → 导出）。详见 [升级指南](UPGRADE.md)。

## 已知限制

- 误判检测基于关键词规则（非 LLM）
- 语义搜索需要 OpenAI API Key
- 导出不含 embeddings（导入后需重建索引）
- 导入为合并模式，不做去重
- Vision 推测的目标用户/价格带为 AI 推断，仅供参考，需用户核对
- Apple Silicon Mac 通过 Docker 理论上支持，但未经真实 Mac 完整测试
